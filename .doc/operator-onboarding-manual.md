# 운영자 매뉴얼 — 사용자 온보딩

> **대상**: 팀장 / 총괄간사 / 간사 (사용자 등록·초대 발송 담당)
> **사전 지식**: 카카오 OAuth 로그인 + 친구 동의 의미 이해
> 빠른 실행은 [first-onboarding-checklist.md](./first-onboarding-checklist.md), 문제 발생 시 [troubleshooting.md](./troubleshooting.md).

## 1. 시스템 구조 요약

- **프론트**: `https://ksea-m.vercel.app` (Vercel 배포)
- **백엔드**: `https://mornitoring-api.onrender.com` (Render 배포)
- **DB**: PostgreSQL (Supabase Seoul)
- **인증**: 이메일+비번 로그인 또는 카카오 OAuth
- **카카오 알림**: 발신 간사의 카카오 토큰으로 친구 메시지 발송 (`send_message_to_friends`)

## 2. 사용자 등록 흐름

### 2-1. 단건 등록

1. `/admin` 진입
2. 우측 상단 **"사용자 등록"** 클릭
3. 입력: 이름 / 이메일 / 역할(팀장·총괄간사·간사·검토위원) / 전화번호(선택)
4. 등록 완료 → 다이얼로그에 **일회용 초기 비밀번호** 표시
5. 비밀번호를 사용자에게 전달 (또는 바로 다음 단계 "초대 발송"으로 비번 재설정)

> 일회용 비밀번호 방식은 과도기 fallback. **권장: 등록 후 "초대 발송"**으로 사용자가 직접 비번 설정하게 함.

### 2-2. 엑셀 일괄 등록

1. 엑셀 양식: A=이름, B=이메일, C=역할(`검토위원` 등 한글 또는 `reviewer` 영어), D=전화번호(선택). 헤더 1행 + 데이터 2행~
2. `/admin` 우측 상단 **"등록 후 자동 초대 발송"** 체크박스 ON 유지 (기본 ON)
3. **"엑셀 일괄등록"** 클릭 → 파일 선택
4. 결과:
   - `등록 N명 / 스킵 M명` 카운트
   - 자동 발송 결과 다이얼로그 (다음 절 참조)
5. 같은 이메일이 이미 있으면 `skipped`로 처리됨 (덮어쓰기 X)

## 3. 카카오 친구 매칭 (선택)

발신 간사의 카카오 친구 목록과 사용자 계정을 1:1 매칭하면 카카오 자동 발송 활용 가능.

1. `/admin` 진입 → 상단 배너에서 카카오 동의 항목 모두 OK 확인 (간사 본인의 카카오 OAuth 완료 + 친구·메시지 동의)
2. **"친구 목록 새로고침"** 클릭 → 발신 간사의 카카오 친구 목록 조회
3. 각 사용자 행에서 **"매칭"** 버튼 클릭 → 친구 선택
4. 매칭 완료 후 **"매칭 변경"** / **"매칭 해제"** 가능

> 매칭 안 된 사용자는 초대 발송 시 **수동 전달용 setup_url**이 화면에 표시됨. 별도 채널(SMS/이메일)로 전달.

## 4. 초대 발송 (비밀번호 셋업)

사용자가 직접 비번을 설정하도록 시간 제한 토큰 기반 링크 발송.

### 4-1. 단건 발송

1. 사용자 행에서 **"초대 발송"** 버튼 클릭
2. confirm: 카카오 매칭 여부에 따라 다른 안내문
3. 응답 분기:
   - **카카오 매칭 + 발송 성공**: alert "카카오 메시지로 초대 링크 발송"
   - **카카오 미매칭 또는 발송 실패**: 다이얼로그에 setup_url 노출 → 복사 → 별도 채널 전달
4. 사용자가 링크 클릭 → `/setup-password?token=...` → 새 비번 입력 → 완료

### 4-2. 일괄 발송 (다중 선택)

1. 사용자 목록 좌측 체크박스로 대상 선택 (헤더 체크박스로 전체 선택 가능)
2. 상단 액션 바의 **"선택한 N명에게 일괄 초대 발송"** 클릭
3. confirm 후 결과 다이얼로그:
   - **요약 카드 4개**: 총 / 카카오 발송 / 수동 전달 / 실패
   - **수동 전달 필요 섹션**: 펼침 + "전체 복사" 버튼 (TSV 형식 `이름\t링크`)
   - **카카오 발송 완료 / 실패 섹션**: 접힘 (필요 시 펼치기)
4. 수동 전달 필요 사용자에게 별도 채널로 링크 전달

### 4-3. 토큰 만료 / 재발송

- **토큰 유효 기간**: 72시간
- **재발송 시**: 같은 사용자의 이전 미소비 토큰은 자동 무효화 → 마지막 발급분만 유효
- 사용자가 링크 만료 안내 받으면 **"초대 발송"** 단건으로 재발송

## 5. 비밀번호 재설정

기존 비번 분실 사용자 대응:

- **방법 A (권장)**: **"초대 발송"** 버튼 — D 토큰 흐름으로 사용자가 직접 새 비번 설정
- **방법 B (과도기)**: **"PW초기화"** 버튼 — 일회용 난수 비번 즉시 발급 → 다이얼로그 복사 → 사용자 전달

> 운영 안정화 후 방법 B는 단계적 제거 예정.

## 6. 사용자 정보 수정 / 삭제

- **수정**: 사용자 행 **"수정"** 버튼 — 이름/전화/역할/활성여부
- **삭제**: 사용자 행 **"삭제"** 버튼 — 비활성화가 아니라 완전 삭제. 신중.

## 7. 운영 모니터링 (현재 가능 범위)

- **카카오 발송 이력**: `/notifications` 페이지 (팀장/총괄간사)
- **본인이 받은 알림**: `/notifications/my` (모든 사용자)
- **감사 로그**: `/api/audit-logs` (팀장/총괄간사 — UI 없음, API 직접 조회)
- **운영 로그**: Render 대시보드에서 `event=` 키워드 검색 (예: `event=auth_login_failed`)

## 8. 운영 정책 요약

전체 정책은 [operations-policy.md](./operations-policy.md). 자주 참조:

- 비번 셋업 토큰: 72h 만료, 재발송 시 이전 토큰 무효화
- 카카오 발신 quota: 30,000건/일 (검수 통과)
- 관리자 권한: 팀장(전부) > 총괄간사(전부) > 간사(읽기 전체 + 일부 쓰기) > 검토위원(본인 담당만)
- 토론방: 모든 인증 사용자 자유 작성
- 공지사항: 작성은 SECRETARY 이상, 댓글은 모든 사용자
