# 검토위원 50명 첫 온보딩 체크리스트

> **목적**: 검토위원 50명을 시스템에 1회 일괄 온보딩.
> **소요 시간**: 약 1~2시간 (사용자 응답 대기 제외).
> 운영 중 펼쳐놓고 그대로 따라간다. 상세 절차는 [operator-onboarding-manual.md](./operator-onboarding-manual.md), 문제 발생 시 [troubleshooting.md](./troubleshooting.md).

---

## 0. 사전 준비 (발송 담당 간사 본인)

- [ ] `https://ksea-m.vercel.app/login`에서 카카오 로그인 완료
- [ ] `/admin` 진입 → 상단 배너에서 **"카카오 동의 항목 모두 OK"** 확인
  - 필수 동의: `profile_nickname`, `friends`, `talk_message`
  - 미동의 항목 있으면 **"추가 동의받기"** 버튼으로 재동의
- [ ] 카카오 일일 발송 quota(30,000건) 여유 확인 — 50명이라 사실상 무관
- [ ] **검토위원 50명과 카카오톡 친구 관계 확보** (각자 친구추가 완료)

## 1. 엑셀 양식 준비

- [ ] 엑셀 양식: A=이름, B=이메일, C=역할(`검토위원` 또는 `reviewer`), D=전화번호(선택)
- [ ] 첫 행은 헤더 (데이터는 2행부터)
- [ ] 50명 명단 + 이메일 주소 정확성 검증 (오타 시 사용자 비번 설정 못 함)
- [ ] 같은 이메일 중복 없음 확인 (있으면 `skipped`로 처리)

## 2. 일괄 등록 + 자동 초대 발송

- [ ] `/admin` 진입
- [ ] **"등록 후 자동 초대 발송"** 체크박스 ON 유지 (기본 ON)
- [ ] **"엑셀 일괄등록"** 클릭 → 엑셀 파일 선택
- [ ] 결과 확인:
  - `등록 N명 / 스킵 M명` 카운트
  - 자동 발송 결과 다이얼로그: 총/카카오발송/수동전달/실패 4 카드

## 3. 카카오 친구 매칭 + 카카오 발송 결과 확인

- [ ] **카카오 발송 완료 사용자**: 추가 작업 없음 — 사용자가 카카오 메시지에서 링크 클릭
- [ ] **수동 전달 필요 사용자** (대부분 카카오 매칭 안 된 신규 사용자):
  - 결과 다이얼로그의 "수동 전달 필요" 섹션 펼침
  - **"전체 복사"** 버튼으로 TSV 형식 복사 (이름\\t링크)
  - SMS·이메일 등 별도 채널로 사용자별 링크 전달
- [ ] **실패 사용자**: 사유 확인 → 재시도 또는 트러블슈팅

## 4. 카카오 매칭 (선택, 첫 발송 직후)

검토위원이 카카오 로그인 후 친구 관계가 형성된 경우, 다음 발송부터는 카카오 자동 발송 활용:

- [ ] **"친구 목록 새로고침"** 클릭
- [ ] 사용자 목록의 각 행에서 **"매칭"** 버튼 → 친구 선택
- [ ] 매칭 후 다음 발송부터 카카오 자동 발송됨

## 5. 사용자 응답 모니터링

- [ ] 발송 후 **24시간 경과 시** 비번 미설정자 식별
  - 현재 UI에는 "비번 설정 완료" 직접 표시는 없음
  - DB 직접 확인: `password_setup_tokens.consumed_at IS NULL` AND `expires_at > now`
  - 또는 사용자별 카카오/전화 확인
- [ ] 미설정자에게 **"초대 발송"** 단건 버튼으로 **재발송**
  - 재발송 시 이전 토큰 자동 무효화 → 새 링크만 유효
- [ ] 발송 후 **72시간 경과 시** 토큰 만료 — 미설정자 일괄 재발송

## 6. 검토위원 계정 ↔ 건물 매핑 백필

검토위원 50명 계정 생성 후, 기존 건물 데이터의 `assigned_reviewer_name`(이름)을 `reviewer_id`(FK)에 매핑:

- [ ] 백엔드에서 dry-run:
  ```bash
  cd backend && source .venv/bin/activate
  python -m scripts.backfill_reviewer_id
  ```
- [ ] 결과 리포트 확인:
  - **매핑 가능**: 단일 후보 사용자
  - **매핑 불가 (User 없음)**: 이름 오타 가능성 — 엑셀과 비교
  - **매핑 불가 (동명이인)**: 수동 처리
  - **매핑 불가 (Reviewer 행 없음)**: 별도 INSERT 필요
- [ ] 문제 없으면 실제 반영:
  ```bash
  python -m scripts.backfill_reviewer_id --apply
  ```

## 7. 운영 시작 후 1주 내 후속

- [ ] AWS IAM 기존 키 `AKIAZZPZKADQH5MH5YMY` Deactivate (보안 잔여)
- [ ] "ksea" 잔존 계정 3명 비번 초기화 — `/admin`에서 "PW초기화" 또는 "초대 발송"
- [ ] 검토위원 비번 미설정자 retry → 100% 활성화
- [ ] 카카오 친구 매칭 100% 완료 → 다음 알림은 모두 카카오 자동

## 실행 후 남겨야 할 기록

운영 추적을 위해 별도 시트(또는 Notion/메일)에 기록 권장:

- [ ] **일괄 발송 일시** (예: 2026-04-18 14:30)
- [ ] **수동 전달 대상자 명단**: 결과 다이얼로그의 "전체 복사" → TSV 보관
- [ ] **수동 전달 채널·일시**: 누구에게 SMS/이메일로 언제 보냈는지
- [ ] **재발송 대상자**: 24h/72h 시점에 미설정자 목록
- [ ] **백필 실행 결과**: matched / unresolved 카운트

이력은 다음 발송·복구 시점에 참조하고, 필요 시 운영 회의 기록으로 활용.
