# 프로그램의 목적

- 건축구조안전 모니터링 업무를 효율적으로 처리하기 위한 통합 솔루션
- 사용자 구성: 모니터링 팀장 1인 + 총괄간사 1인 + 간사 5인 + 검토위원 50명 ≈ 약 60명 (숫자는 변경 가능)

# 시스템 구성

- **Frontend**: Next.js 16 (App Router) + TypeScript + Tailwind + shadcn/ui
- **Backend**: FastAPI + SQLAlchemy + Alembic + Pydantic v2
- **DB**: PostgreSQL 17 (Supabase)
- **Storage**: AWS S3 (검토서 파일, 첨부파일)
- **인증**: JWT + 카카오 OAuth2
- **알림**: 카카오톡 친구 메시지 API + 나에게 보내기 (2026-04-17 검수 통과)
- **배포**: Frontend → Vercel (ksea-m.vercel.app) / Backend → Render

# 업무 프로세스

### Phase 0: 관리번호 부여 + 검토위원 배정
- 관리번호 형식: `YYYY-NNNN` (예: 2026-0001)
- 통합관리대장(엑셀)에 검토위원 열을 총괄간사가 기입 → 웹에 엑셀 업로드
- 배정 완료 시 건물 `current_phase`가 `assigned`로 설정

### Phase 1: 설계도서 접수/배포
- 폴더 단위로 접수 (폴더명에 관리번호 포함)
- 웹에서 관리번호 목록을 입력하면 **차수 자동 판별** (예비/보완 1~5차)
- 검토위원에게 카카오톡 알림 (수동 발송 버튼)
- 검토위원은 `[내 검토 대상]` 페이지에서 확인 (설계도서는 별도 웹하드에서 다운로드)

### Phase 2: 검토서 업로드 + 검증
- 검토위원이 `검토서(.xlsm)` 파일 업로드
- **유효성 검증**:
  - 파일명 = 관리번호 (`.xlsm` 전용)
  - 내부 C4(관리번호) = 파일명의 관리번호
  - F4(검토자) = 로그인 사용자
  - 시트명/절차(C5): 예비는 "1차" / 보완은 차수 무관 "2차"
  - H4(검토결과)는 적합/단순오류/재계산 중 하나
  - F12(내진등급)는 특/I/II 중 하나
  - F13(도면작성자 자격)은 건축사/건축구조기술사/기타 중 하나
- **미리보기 → 확인 → 업로드** 2단계 플로우
- 업로드 시 자동 추출 → `review_stages` 반영, 건축물 정보도 동기화
- 같은 단계 재업로드 시 **"기존 검토서 삭제됩니다"** 경고 다이얼로그

### Phase 3: 부적합 대상 검토
- 검토위원이 업로드 시 `부적정 사례 검토 필요` 체크 가능
- 간사 이상이 `/inappropriate-review` 페이지에서 판정
  - **확정(심각) / 확정(단순) / 대기 / 제외** 4단계
  - 간사진 의견을 여러 건 작성 가능 (작성자/작성일 기록)

### Phase 4: 반복 (보완 5차까지)
- `예비검토 → 보완검토(1차) → ... → 보완검토(5차) → 완료`
- 최종 완료(`completed`)는 **별도 엑셀 업로드로 지정 예정** (현재 미구현)
- 각 보완도서 접수는 웹에서 관리번호만 입력하면 차수 자동 판별
- 간사 이상은 건물 상세 페이지에서 **단계 수동 수정** 가능

# 접속 화면 및 기능

## 대시보드 (`/dashboard`)
- **상단 위젯 2×2** (모든 로그인 사용자):
  - 공지사항 최신 5건 | 내가 받은 카톡 알림 5건
  - 토론방 최신 5건 | 나의 문의사항 5건
- **내 담당 현황**: 배정 / 제출된 검토서(예비/보완) / 연면적 / 1,000㎡↑ / 고위험
- **검토 대상**: 검토서 미제출 건수 + 접수 후 경과일수 버킷(1~7일, 1주, 2주이상)
- **전체 현황** (간사 이상): 미배정/배정완료/검토서대기/검토진행중/완료
- **검토위원별 현황** (간사 이상)

# 주요 기능 상세

## 카카오 알림 (검수 통과 완료)
- OAuth 로그인 + 동의 항목 자동 체크 + 재동의 리다이렉트
- 알림 발송: 간사가 수신자 다중 선택 → 수동 버튼 클릭 (자동 발송 없음)
- **본인에게 발송**: "나에게 보내기" API 자동 분기 (친구 관계 불필요)
- pair(발신자↔수신자) 일간 20건 제한 자체 모니터링

## 공지사항
- 작성(간사 이상) + 수정(본인 또는 팀장/총괄간사) + 삭제
- 댓글 (전체 로그인 사용자)
- 첨부파일 다중 (작성 시 + 상세 페이지에서 추가)

## 토론방
- 전체 사용자 작성 가능
- 댓글 + 첨부파일 + 수정/삭제(본인 또는 팀장/총괄간사)

## 부적합 대상 검토
- 4단계 판정 + 다중 의견(작성자 기록)
- 지적단계 표시(예비검토 / 보완검토 N차)
- 필터: 전체/대기/확정(심각)/확정(단순)/제외

## 사용자 관리 + 카카오 매칭 (통합)
- 사용자 CRUD + 엑셀 일괄 등록
- 카카오 로그인 상태 + 친구 매칭 상태 한눈에
- 진단 버튼으로 특정 사용자의 동의 scope 조회

# 보안·운영 규칙

- 로그인 시 JWT 발급, 비밀번호는 bcrypt 해시
- 최초 로그인 시 비밀번호 변경 강제 (기본 `ksea`)
- 감사 로그(audit_logs)에 create/update/delete/upload/assign 등 기록
- S3 파일은 presigned URL로 접근 (만료 1시간)
- 재업로드 시 기존 S3 파일 자동 삭제 (orphan 방지)
