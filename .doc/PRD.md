- 건축구조안전 모니터링 업무를 효율적으로 하기 위한 통합 솔루션
- 이 프로그램의 사용자는 모니터링 팀장 1인, 총괄간사 1일, 간사 5인, 검토위원 50명 으로 구성됨 (숫자는 변경될 수 있음)

# 업무 프로세스

### Phase 0 : 관리번호 부여 및 검토위원 배치

- 관리번호는 각 건물에 대해 국토안전관리원에서 지정함. 
- 관리번호 형식 2026-xxxx (4자리 숫자)
- 각 관리번호에 대응하는 엑셀형식의 "통합관리대장"이 있음
- 총괄간사는 "통합관리대장"에 검토위원 열을 형성하여 검토위원을 수기로 지정함.

### Phase 1 : 설계도서 접수 및 배포 단계

- 설계도서는 폴더 단위로 들어옴.
- 폴더명 앞에는 관리번호가 기입됨 ( 예 : 2026-0001_서울시 송파구...)
- 설계도서는 총괄간사의 컴퓨터에 저장될 예정임.
- 폴더 : D:/2026모니터링/01.접수자료/예비검토/ 아래 하위 폴더로 접수일로 구분해서 배치할 예정 (예시 : D:/2026모니터링/01.접수자료/1. 예비검토/20260501/)
- 총괄간사는 [자료분류] 버튼을 누르면 '폴더 선택'을 할 수 있으며 '저장 폴더'도 지정할 수 있음. 이를 통해 "통합관리대장"의 '저장 폴더'의 하위 폴더에 검토위원 이름으로 된 폴더를 생성하고 관련 자료를 해당 폴더에 옮기는 작업을 자동화함
- 검토위원에게 검토대상이 추가되었다고 카카오톡 API를 이용해서 알림을 받음.
- 검토위원은 로그인을 하면, [검토대상]을 확인할 수 있음. (설계도서는 별도의 웹하드에서 다운을 받을 것임. 보안으로 인해 다른 웹상에서 저장되면 안됨)

### Phase 2 : 도서 검토 및 검토서 제출 단계

- 검토위원은 설계도서를 확인하여 "검토서"를 작성함. "검토서"는 단일시트의 엑셀로 구성되어 있음.
- 검토위원은 이 프로그램을 이용하여 "검토서"를 업로드함. 
- "검토서"가 업로드 되면 다음과 같은 유효성 검증 작업이 수행되어야 함.
  - 검토서의 파일명은 관리번호로만 구성됨. (예 : 2026-0001.xlsm)
  - 검토서 내의 관리번호와 파일명은 일치되어야 함.
  - 제출한 사람의 이름이 검토자 이름에 기입이 되어야 함.
  - 유효성 검증 시 문제가 있으면 이를 사용자에게 알리고 수정 후 다시 올리도록 하여야 함.
- 유효성 검증이 완료된 "검토서"는 임의의 폴더 (google drive 이용 또는 AWS이용)에 제출된 날짜 폴더에 저장함.
- 저장된 검토서의 내용을 자동으로 추출하여 "통합관리대장"에 내용을 기입함.

### 패턴 반복

- 모니터링 업무는 다음과 같은 단계를 거침
  - 예비검토 -> 보완검토(1차) -> 보완검토(2차) -> 보완검토(3차) -> ...
